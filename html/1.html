Practical introduction to the Windows code signing

The X.509 is the most popular standard defining the format of public key certificates. It is also the only one, I personally know.

According to the Wikipedia - “An X.509 certificate binds an identity to a public key using digital signature”. Based on this statement we can deduct, that the main purpose for the X.509 certificates is to verify / authenticate the identity (organization or person), which issued the certificate.

The certificates can confirm the author of any digital artifact you’re using - website, firmware, documents or even programs.
Thanks to it, we’re able to decide, who we trust.
